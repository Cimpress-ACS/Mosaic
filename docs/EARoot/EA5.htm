MEF is used for module base approach. MEF works with C# attributes, so it is not visible in this diagram.<br /></p><p>But nearly each interface in this diagram uses MEF Export or Import.<br /></p><p><br /></p><p>Typical MEF parts are:<br /></p><p><ul><li>ILogger</li><br /></p><p><li>IEventAggregator</li><br /></p><p><li>ITagListener</li><br /></p><p><li>ITagController</li><br /></p><p><li>ITagImporter</li><br /></p><p><li>IItemManager</li><br /></p><p><li>PlatformModule</li><br /></p><p></ul><br /></p><p>The MefBootstrapper loads all parts recursively from a root directory. The PlatformModuleInitializer is used at startup to wire up the modules together. It's like a double linked list because each module hold references to previous and next (except source and sink).<br /></p>
